## Moments forts

- **Le seuil 5px** pour le drag mobile : subtil mais important. Sans lui, taper sur un bouton de mode pouvait déclencher accidentellement un drag. Détecté lors de la lecture du research/37.
- **Le `create()` async dans BootScene** : nécessaire pour `await PortalSDK.init()` (SDK Poki = async). Phaser 4 le supporte nativement.
- **PortalSDK.js zero dépendance externe** : les vrais SDK ne sont injectés que via VITE_PLATFORM au build. En dev local, le jeu tourne sans rien toucher.

## Décisions notables

- Dust limit **5 desktop / 4 mobile** (pas 3/3 comme prévu initialement) : conserver l'effet visuel, limiter la pression GC mobile.
- `IS_MOBILE` via `navigator.userAgent` : plus robuste qu'une détection par taille d'écran (qui créerait de faux positifs en fenêtre réduite desktop).
- **Pas de SDK CrazyGames/Poki chargés** — règle stricte documentée dans PortalSDK.js. Les portails seront intégrés uniquement au moment de la soumission.
