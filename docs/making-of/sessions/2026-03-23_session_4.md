## Moments forts
- **La revelation PixelLab** : en donnant une boule chrome lisse comme reference de style et un prompt "16 couleurs differentes", PixelLab genere une planche parfaite de 16 boules coherentes. Le joueur realise que l'hyper-personnalisation est possible a moindre cout — c'est un game changer pour un dev solo
- **Le probleme de lisibilite** : les premieres boules (v2) avaient des motifs graves magnifiques en preview 64x64, mais illisibles a 20px en jeu. Ca a force a repenser la strategie : boules simples pour le gameplay, detaillees pour les menus
- **3 iterations en une session** : v1 (lisses 3D) → v2 (motifs graves detailles) → v3 (simples avec variations subtiles). Chaque iteration a enseigne quelque chose

## Decisions notables
- **Boules simples > boules detaillees** pour le gameplay a petite echelle. Les motifs complexes deviennent du bruit visuel a 20px
- **Mix de styles** : plutot qu'un style unique, chaque couleur a son propre style (pure/ligne/courbe/motif/lisse) pour de la variete dans la simplicite
- **16 couleurs** au lieu de 10 — plus de personnalisation possible, nouvelles couleurs thematiques (lavande = Provence, obsidienne = puissance, corail = calanques)
- **Systeme boutique simplifie** : une seule categorie au lieu de 4 sous-collections, prix de 0 (gratuit) a 500 galets
- **Option D retenue** pour le futur : sprites detailles pour les menus, simples pour le jeu (hybrideimplementable plus tard)

## Etat visuel
16 boules de couleurs variees dans le selecteur de partie rapide. Chaque boule tourne quand elle roule sur le terrain. Preview dans le panneau droit avec stats et lore humoristique. La boutique affiche les 16 boules avec leurs prix.

- **Apres** : 16 boules pixel art distinctes, chacune avec son style, animation de rotation 6 frames, 6 nouvelles couleurs, lore unique par boule
