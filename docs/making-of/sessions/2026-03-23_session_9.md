## Moments forts
- La grande aventure des bouts de rondins : 2h a essayer toutes les rotations/flips possibles pour placer 8 sprites de coins. On a fini par creer une page HTML interactive ou l'utilisateur peut cliquer pour changer l'orientation de chaque bout de rondin, puis coller le resultat. Solution finale : pre-baker les 8 images avec les transforms exactes — zero rotation dans Phaser.
- La decouverte que les transforms CSS et Phaser ne fonctionnent PAS pareil (ordre d'application different), ce qui expliquait pourquoi l'editeur HTML marchait mais pas le jeu.
- Generation PixelLab de 3 tilesets supplementaires en une seule session — le pipeline "generer sur PixelLab → decouper → extraire bandes 16px → implementer" est rode.

## Decisions notables
- **Pre-baking plutot que transforms runtime** : au lieu de setAngle()+setFlipX()+setFlipY() dans Phaser (fragile et non-intuitif), on genere les PNG deja tournes. Zero maths, zero bugs.
- **Bordures H couvrent les coins** : les tileSprites horizontaux font tw+bw*2 de large (incluant les coins), les verticaux font juste th. Pas de gaps possibles.
- **Display scale vs physics radius** : pour reduire la taille visuelle des boules sans casser la physique, on a ajoute BALL_DISPLAY_SCALE (0.85) et COCHONNET_DISPLAY_SCALE (0.70) qui multiplient le scale du sprite sans toucher au rayon de collision.
- **Ficelle realiste** : les dead lines sont maintenant une ficelle 1px avec leger wobble et piquets aux coins, exactement comme en competition de petanque reelle.
